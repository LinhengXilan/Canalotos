对ChangeLog的说明

1. ChangeLog使用markdown编写  
<br/>
2. ChangeLog应按照以下格式编写  


    commit：commit哈希  
    author(s)：没写就是我
    build号：build提交日期（yyyy-mm-dd）
    修改标题  
    file：所修改的主要文件（以分号分隔，可多行，以项目根目录起始）  


        对修改的说明

3. 修改类型  

   A: 添加  
   D: 删除  
   F: 修复  
   M: 修改  
   O: 优化


4. 每个小版本编写一个ChangeLog  


5. ChangeLog应及时编写
